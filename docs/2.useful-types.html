Good for doing unions, intersections, etc of data sets. ```python core = set('tomatillos', 'serrano chile', 'garlic', 'onion') garnish = set('onion', 'cilantro', 'kosher salt') salsa_verde = core.union(garnish) print(salsa_verde) ``` ```console >>> {'tomatillos', 'serrano chile', 'garlic', 'onion', 'cilantro', 'kosher salt'} ``` ```python salsa_verde = core | garnish print(salsa_verde) ``` ```console >>> {'tomatillos', 'serrano chile', 'garlic', 'onion', 'cilantro', 'kosher salt'} ``` ```python redundant_ingredients = core & garnish print(redundant_ingredients) ``` ```console >>> {'onion'} ``` --- class: center, middle # Neverending arguments --- # Neverending arguments "Variadic" functions accept a variable number of arguments. ```python def sum_however_many_args( *args ): print(args) print(type(args)) # tuple # add them all up total = 0 for arg in args: total += arg return total ``` Try it yourself: ```python result = sum_however_many_args(1, 2, 3, 4, 5) print(result) ``` --- # Neverending arguments "Variadic" functions can require positional args. ```python def sum_at_least_two_args(a, b, *args ): print(args) print(type(args)) # tuple # add them all up total = a + b # start with the required args for arg in args: total += arg return total ``` Try it yourself: ```python result = sum_at_least_two_args(1, 2, 3, 4, 5) print(result) ``` --- # Neverending arguments An arbitrary number of named arguments can be accepted into a dictionary parameter. ```python def g( **kwargs ): print(kwargs) ``` Try it yourself: ```python g(a=1, b=2, c=3) ``` ```console >>> {'a': 1, 'b': 2, 'c': 3} ``` --- # Neverending arguments And of course, with keyword args, it's possible to require a few positional arguments, neverending anonymous arguments, in addition to limitless, keyword args. ```python def crazy(a, b, *args, **kwargs): print(a, b, args, kwargs) ``` Try it yourself: ```python crazy(1, 2, 3, 4, z=1) ``` ```console >>> 1 2 (3, 4) {'z': 1} ``` --- class: center, middle # Unpacking unpacking --- # Unpacking unpacking The `*` before an an argument to a function call that is an iterable will break up, or unpack, the values into discrete values as separate arguments. ```python x = ['Twas', 'brillig', 'and', 'the', 'slithy', 'toves'] print(*x) ``` This is what is *not* printed (but would have been without the `*`): ```console >>> ['Twas', 'brillig', 'and', 'the', 'slithy', 'toves'] ``` It's as if each value in the list was passed as a discrete argument to the print function. ```console >>> Twas brillig and the slithy toves ``` --- class: center, middle # Functions of the finest order --- # Functions of the finest order Functions can be passed around the table, like any other data. An innocent function: ```python def bar(): print('hello') ``` A not-so-innocent function: ```python def foo(func): func() ``` Use the second function to call the first ```python foo(bar) ``` This can be useful in some design patterns. --- class: center, middle # Decorators --- # Decorators Not just to make your windows pretty. Imagine this function ```python def call_it(func, n): def new_func(x): res = x for i in range(n): res = func(res) return(res) return new_func ``` --- # Decorators Functions that decorate othere functions. And this innocent function ```python def double(num): return num * 2 ``` Triggered by another function ```python my_func = call_it(double, 3) print(my_func(2)) ``` This pattern is called the Decorator pattern. The `call_it` function decorates, or adds features and functionality to the `double_num` function that don't exist in the original. --- class: center, middle # Conclusions --- # Conclusions Thank you.
